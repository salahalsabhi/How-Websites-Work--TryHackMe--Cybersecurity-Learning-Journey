# How Websites Work - TryHackMe

## Overview

This repository documents my completion of the **How Websites Work** room on TryHackMe. The room provides a solid foundation in web technologies, explaining how websites function, how browsers interact with web servers, and how common web vulnerabilities arise from insecure development practices.

Understanding how websites work is a critical first step toward web application security and penetration testing.

## Skills Learned

### Web Fundamentals

* Understanding the client-server model
* How browsers communicate with web servers
* Website architecture and components
* Frontend vs Backend technologies

### HTTP & Web Communication

* HTTP requests and responses
* Common HTTP methods (GET, POST)
* Status codes and their meanings
* URLs, domains, and web resources

### Website Technologies

* HTML fundamentals
* CSS basics
* JavaScript fundamentals
* How web pages are rendered in browsers

### Security Concepts

* Sensitive Data Exposure
* HTML Injection
* Input Validation & Sanitization
* Importance of reviewing page source code
* Understanding hidden content and developer mistakes

## Key Takeaways

* Never trust user input.
* Sensitive information should never be exposed in frontend code.
* Proper input sanitization is essential for secure web applications.
* Understanding web technologies is fundamental before learning web exploitation techniques.
* Viewing page source and analyzing frontend code can reveal valuable information during security assessments.

## Tools & Techniques Covered

* Browser Developer Tools
* View Page Source
* HTML Inspection
* Basic Web Reconnaissance
* Understanding Client-Side Functionality
